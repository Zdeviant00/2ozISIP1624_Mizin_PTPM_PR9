## 2. WPF Приложение
*Раздел в процессе*

### Структура проекта
- **Проект:** `Mizin_WPF_PR9`
- **Папка страниц:** `Pages/` (AuthPage.xaml, RegPage.xaml)
- **Модель данных:** `Model1.edmx` (Entity Framework 6.x)
- **Класс контекста БД:** `Mizin_23ISIP1624_Entities`
- **Подключение к БД:** через `App.config`

### Страницы приложения
1. **MainWindow** - главное окно с кнопкой "Назад" и Frame для навигации
2. **AuthPage** - страница авторизации (логин + пароль)
3. **RegPage** - страница регистрации нового пользователя

### Навигация
- `MainWindow` содержит `Frame` с свойством `Source="/Pages/AuthPage.xaml"`
- При запуске приложения автоматически загружается страница авторизации
- Кнопка "Назад" реализует возврат по истории навигации

### Функционал
### Страница авторизации (AuthPage)
- Поля: Логин, Пароль
- Кнопки: Вход, Регистрация
- Проверки: заполненность полей, существование пользователя, проверка пароля, проверка на "Удален"
- Приветствие: "Здравствуйте, {Роль} {ФИО}!"
- Авторизация пользователей с проверкой в БД
![Auth1](https://github.com/user-attachments/assets/04502a93-6e5b-4368-ae8d-fd5d1da835ea)

### Страница регистрации (RegPage)
- Поля: ФИО, Логин, Пароль, Подтверждение пароля
- Кнопки: Регистрация, Отмена
- Проверки:
  - Заполненность всех полей
  - Уникальность логина в БД
  - Сложность пароля (6+ символов, английская раскладка, цифры)
  - Совпадение паролей
- По умолчанию роль: "Менеджер А"
- Новый пользователь сохраняется в БД

- Регистрация с валидацией:
  - Проверка заполнения всех полей
  - Проверка уникальности логина
  - Проверка сложности пароля (6+ символов, английская раскладка, цифры)
  - Проверка совпадения паролей
- Приветствие с ролью пользователя после входа

### Рефакторинг для тестирования
- Метод `Auth(string login, string password)` вынесен из обработчика кнопки
- Метод `Register(string fio, string login, string password, string confirmPassword)` вынесен
- Оба метода возвращают `bool` — результат операции
- Доступны для вызова из модульных тестов
